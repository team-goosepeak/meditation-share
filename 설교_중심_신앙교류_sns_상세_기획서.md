# 설교 중심 신앙교류 SNS — 상세 기획서

## 1. 개요
- **서비스명(임시)**: SermonShare (한국어 예시: 말씀나눔, 고백플랫폼)
- **목표**: 교회 예배 중 설교 말씀을 중심으로 개인의 묵상과 신앙 고백을 기록하고, 교회 내외 크리스천들이 서로의 신앙 생활을 공감·격려하며 영적 성장을 돕는 소셜 플랫폼을 제공한다.
- **핵심가치**: 말씀중심성, 공감/비판금지(비판 없는 안전한 고백), 소그룹 연결(교회·셀 단위), 꾸준한 묵상 습관 형성

---

## 2. 핵심 사용자 페르소나
1. **청년부 리더 민수(25)**
   - 특징: 대학생/직장초년생, 스마트폰 활발 사용, 묵상 습관 만들고 싶음
   - 니즈: 설교를 들은 후 즉시 메모하고 공유, 같은 교회 친구들과 토론하고 싶음

2. **중년 성도 지혜(45)**
   - 특징: 직장/가정으로 바빠 짧게 묵상하고 싶음
   - 니즈: 핵심 구절 북마크, 기도 요청 기능, 개인 신앙 기록 보관

3. **목회자/전도사(38)**
   - 특징: 설교자 관점에서 피드백과 교인 반응을 알고 싶음
   - 니즈: 설교 피드백(익명 선택 가능), 교회 그룹 운영 툴

---

## 3. 핵심 기능 (MVP 우선순위 포함)
### MVP (필수)
1. **설교 기록 포스트 작성**
   - 필드: 제목(자동: 교회명+날짜+설교제목 선택 가능), 본문(묵상/고백), 관련 성경구절(검색/입력), 예배일시, 예배처(교회/온라인), 태그(감정·주제)
   - 공개범위: 전체공개 / 교회(그룹) / 친구 / 비공개
2. **피드(타임라인)**
   - 팔로우 기반 또는 교회 기반 필터
   - 하이라이트(오늘의 말씀, 추천 포스트)
3. **댓글/응원(에모지 기반)**
   - 공감(하트·기도·아멘 등), 댓글(신앙 고백 나눔 권장), 답글
   - 비판금지 안내 문구 및 신고 기능
4. **교회/그룹 기능**
   - 교회 생성/가입, 교회 내 공지·설교 목록 접근(목회자 권한)
5. **온보딩 + 첫 묵상 템플릿 제공**
   - 초보자용 질문 가이드(요약, 느낀 점, 적용점, 기도제목)
6. **알림(푸시/앱 내)**
   - 댓글/응원/교회 공지 등

### 2차(우선순위 낮음 — 초기 확장)
1. **설교 오디오/영상 업로드 & 연동(또는 외부 링크)**
2. **주제별 큐레이션(예: 용서, 감사, 가정)**
3. **일별/주별 묵상 챌린지 및 리마인더**
4. **프라이빗 셀(소그룹) 채팅 및 정기모임 스케줄링**
5. **목회자 전용 대시보드(교인 참여 통계)**
6. **익명 고백 모드(신분 보호)**
7. **기도제목 라이브 피드 + 응답 트래킹**

---

## 4. 주요 UX 흐름 (사용자 시나리오)
1. **첫 방문(온보딩)**
   - 교회 선택(직접 검색 또는 교회 초대 링크)
   - 관심 주제 선택(예: 신앙성장, 가정, 직장) — 피드 개인화
   - 묵상 템플릿 소개(간단 튜토리얼)

2. **설교 들은 후 즉시 기록**
   - 홈 → ‘새 묵상 작성’ → 예배정보 자동 추천 → 본문 작성 → 공개범위 설정 → 게시

3. **다른 사람 묵상 읽기 & 교류**
   - 피드에서 포스트 열람 → 공감/기도/댓글 → 저장(북마크)

4. **교회 관리자(목회자) 사용 흐름**
   - 교회 생성 → 멤버 초대 → 설교 업로드/링크 첨부 → 교회 공지 발송

## 5. 기능별 상세 사양 (개발자에게 전달할 수준)
### 포스트 모델 (예시 JSON)
```
{
  "postId": "uuid",
  "authorId": "user-uuid",
  "churchId": "church-uuid|null",
  "title": "2025-10-23 - 사랑의 본질",
  "body": "오늘 설교에서...",
  "scriptures": [{"book":"요한복음","chapter":3,"verseFrom":16,"verseTo":18}],
  "tags": ["용서","감사"],
  "visibility": "public|church|friends|private",
  "createdAt": "ISO8601",
  "updatedAt": "ISO8601",
  "likes": {"count": 10},
  "commentsCount": 3
}
```

### API 엔드포인트(요약)
- `POST /api/posts` — 새 묵상/설교 포스트 작성
- `GET /api/posts` — 피드(필터: 교회, 팔로우, 태그, 날짜)
- `GET /api/posts/{id}` — 포스트 상세
- `POST /api/posts/{id}/comments` — 댓글
- `POST /api/churches` — 교회 생성
- `POST /api/users/{id}/join-church` — 교회 가입
- `POST /api/auth` — 로그인/회원가입(소셜 로그인 지원 토큰)

### DB 구조 제안
- Users (userId, displayName, email, churchIds[], profile, role)
- Churches (churchId, name, address, pastorId, joinCode)
- Posts (위 JSON)
- Comments (commentId, postId, authorId, body, createdAt)
- Reactions (userId, postId, type)
- Notifications

### 권한 모델
- Role: member / pastor / admin
- 목회자 권한: 교회 공지, 설교 업로드, 멤버 관리
- 게시물 가시성 제어: 교회게시 시 해당 교회 멤버만 열람 가능
